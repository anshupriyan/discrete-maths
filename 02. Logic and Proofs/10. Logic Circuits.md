# 1.2.6 Logic Circuits

Propositional logic can be applied directly to the design of computer hardware. This ground-breaking connection was first observed in 1938 by the legendary computer scientist and mathematician Claude Shannon in his MIT master’s thesis.

---

### Basic Building Blocks: Logic Gates

A **logic circuit** (or digital circuit) receives one or more input signals, $p_1, p_2, \dots, p_n$, each representing a single **bit**. In computer circuitry, this bit is fundamentally an electrical state:
* **`1`** represents **On** (or True).
* **`0`** represents **Off** (or False).

The circuit processes these incoming signals to produce an output bit sequence. While complex systems utilize multiple output lines, we focus on circuits that yield a **single output signal**.

Any digital system, no matter how massive or complex, is constructed using combinations of just **three basic circuits called gates**:

#### 1. The Inverter (NOT Gate)
* **Behavior:** It takes a single input bit $p$ and flips its value to its exact logical opposite.
* **Logical Operator:** Negation ($\neg p$).
* **Rule:** If input is `1`, output is `0`. If input is `0`, output is `1`.

#### 2. The OR Gate
* **Behavior:** It takes two input bits, $p$ and $q$, and produces an output signal.
* **Logical Operator:** Disjunction ($p \lor q$).
* **Rule:** The output is `1` if *at least one* of the input bits is `1`. It only outputs `0` if both inputs are completely `0`.

#### 3. The AND Gate
* **Behavior:** It takes two input bits, $p$ and $q$, and binds them together.
* **Logical Operator:** Conjunction ($p \land q$).
* **Rule:** The output is `1` *only* if both inputs are simultaneously `1`. It outputs `0` otherwise.

### Tracing a Circuit to Find its Output Formula

When you are given an existing circuit diagram built from these three gates, you can determine its final algebraic logic formula by systematically **tracing through the gates from left to right**.

Let’s look at the textbook layout for a standard combinatorial circuit:

#### Textbook Example 10 Walkthrough

Let's find the logical expression for a circuit with three input bits ($p, q, \text{ and } r$) configured as follows:
1. Input $q$ passes straight into an **Inverter**, producing **$\neg q$**.
2. This **$\neg q$** and the original input **$p$** are fed directly into an **AND Gate**. The intermediate output of this gate is **$p \land \neg q$**.
3. Separately, input $r$ passes through an **Inverter**, producing **$\neg r$**.
4. Finally, the intermediate result (**$p \land \neg q$**) and the inverted signal (**$\neg r$**) are combined through a final **OR Gate**.

$$\text{Final Output Formula: } (p \land \neg q) \lor \neg r$$

### Designing a Circuit from a Formula

In hardware engineering, you usually have to do the exact opposite: you start with a mathematical system requirement and need to design the physical logic circuit to produce it.

#### Textbook Example 11 Walkthrough

Let’s systematically build a digital circuit that outputs this exact boolean target formula:

$$(p \lor \neg r) \land (\neg p \lor (q \lor \neg r))$$

To construct this cleanly, we split our target into two separate circuit sub-blocks and bridge them together with a final gate at the very end:

* **Sub-Block 1: Producing $(p \lor \neg r)$**
  * Invert input $r$ to get $\neg r$.
  * Feed input $p$ and $\neg r$ into an **OR Gate**.

* **Sub-Block 2: Producing $(\neg p \lor (q \lor \neg r))$**
  * We need the inner piece first: connect $q$ and our inverted $\neg r$ to an **OR Gate** to yield $(q \lor \neg r)$.
  * Invert input $p$ to get $\neg p$.
  * Run $\neg p$ and our intermediate $(q \lor \neg r)$ through another **OR Gate**.

* **The Final Bridge:**
  * Route the output wires from Sub-Block 1 and Sub-Block 2 directly as inputs into a final **AND Gate**.

**Question:** If you are given the input bit states **$p = 1$, $q = 1$, and $r = 0$**, what would the final output value (`1` or `0`) be for the circuit we analyzed in Example 10:

$$(p \land \neg q) \lor \neg r$$

### 💡 Solutions & Explanation

> [!NOTE]
> Here is the step-by-step arithmetic verification:
> 
> * **Final Output Value:** **`1`** (True)
> * **Step-by-Step Evaluation:**
>   1. **Substitute inputs:** $(1 \land \neg 1) \lor \neg 0$
>   2. **Evaluate inverters first:**
>      * $\neg q = \neg 1 = 0$
>      * $\neg r = \neg 0 = 1$
>      * This leaves us with: $(1 \land 0) \lor 1$
>   3. **Evaluate parentheses (AND Gate):**
>      * $1 \land 0 = 0$
>      * This leaves us with: $0 \lor 1$
>   4. **Evaluate final connective (OR Gate):**
>      * $0 \lor 1 = \mathbf{1}$
