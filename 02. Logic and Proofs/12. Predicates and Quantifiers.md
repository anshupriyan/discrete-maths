# 1.4 Predicates and Quantifiers

### 1.4.1 Introduction

Propositional logic (which we studied in Sections 1.1–1.3) is an excellent foundation, but it cannot adequately express the meaning of all statements in mathematics and in natural language.

To see why, suppose we have the statement:
* *“Every computer connected to the university network is functioning properly.”* 

If we are told that *“MATH3 is a computer connected to the university network,”* no rule of propositional logic alone allows us to officially conclude that *“MATH3 is functioning properly.”* Why? Because propositional logic treats each sentence as a single, indivisible atomic block (like $p$ or $q$). It cannot look *inside* sentences to understand the relationship between a general rule ("Every computer") and a specific object ("MATH3").

Similarly, we cannot use the rules of propositional logic to pass from a specific statement like *“CS2 is under attack by an intruder”* to a broader existential conclusion like *“There is a computer on the university network that is under attack by an intruder.”* 

To bridge this gap, we introduce a more powerful, expressive type of logic called **predicate logic** (or **predicate calculus**).

### The Core Upgrades of Predicate Logic

1. **Expressing Meanings Deeply:** It allows us to express the meaning of a wide range of statements in mathematics and computer science in ways that let us reason about individual objects and their traits.
2. **Exploring Relationships:** It lets us seamlessly analyze and explore relationships between distinct elements or data objects.
3. **Quantifying Extent:** It introduces **quantifiers**, which enable us to reason with statements that assert a property holds for *all* objects of a certain type, or that *at least one* object exists with that particular property.

---

## 1.4.2 Predicates

In Section 1.4.1, we discovered that propositional logic falls short because it cannot look inside a sentence to understand its individual components. To resolve this, we break declarative sentences down into two distinct structural parts: a **subject** and a **predicate**.

---

### Anatomy of a Predicate Statement

Consider the statement: *“$x$ is greater than 3.”* 

1. **The Subject:** The variable $x$ is the subject of the statement. It represents an unknown object or value that can change depending on context.
2. **The Predicate:** The phrase *“is greater than 3”* is the predicate. It refers to a specific property or characteristic that the subject can possess.

We formally represent this complete statement as **$P(x)$**, where:
* **$P$** denotes the predicate function (the property "is greater than 3").
* **$x$** is the input variable acting as the subject.

> [!IMPORTANT]
> **Crucial Rule:** On its own, the expression $P(x)$ is **neither true nor false** because $x$ is just an unbound placeholder variable. It is known as a **propositional function**. It only transforms into a true or false **proposition** once a specific value is assigned to $x$.

---

### Single-Variable (Unary) Predicate Examples

#### Textbook Example 1: Numerical Evaluation
Let $P(x)$ denote the statement *“$x > 3$.”* What are the truth values of $P(4)$ and $P(2)$?
* **Evaluating $P(4)$:** We substitute $x = 4$ into the function, yielding the statement *“$4 > 3$.”* This is factually correct, so $P(4)$ is **True ($\text{T}$)**.
* **Evaluating $P(2)$:** We substitute $x = 2$, yielding the statement *“$2 > 3$.”* This is incorrect, so $P(2)$ is **False ($\text{F}$)**.

#### Textbook Example 2: Computer Network Status
Let $A(x)$ denote the statement *“Computer $x$ is under attack by an intruder.”* Suppose that out of all the devices on campus, only `CS2` and `MATH1` are currently compromised. What are the truth values of $A(\text{CS1})$, $A(\text{CS2})$, and $A(\text{MATH1})$?
* **$A(\text{CS1})$:** Since `CS1` is safe and not on the active attack list, the statement is **False ($\text{F}$)**.
* **$A(\text{CS2})$:** Since `CS2` is on the list of compromised systems, the statement is **True ($\text{T}$)**.
* **$A(\text{MATH1})$:** Since `MATH1` is actively under attack, the statement is **True ($\text{T}$)**.

---

### Multivariable ($n$-ary) Predicates

Statements in computer science and mathematics routinely involve more than one variable. A propositional function can accept an arbitrary number of inputs, structurally denoted as $P(x_1, x_2, \dots, x_n)$. This is called an **$n$-place predicate** or an **$n$-ary predicate**.

#### Textbook Example 3: 2-Variable Equation
Let $Q(x, y)$ denote the mathematical statement *“$x = y + 3$.”* What are the truth values of $Q(1, 2)$ and $Q(3, 0)$?
* **Evaluating $Q(1, 2)$:** Set $x = 1$ and $y = 2$. This produces the statement *“$1 = 2 + 3$,”* which simplifies to $1 = 5$. This is **False ($\text{F}$)**.
* **Evaluating $Q(3, 0)$:** Set $x = 3$ and $y = 0$. This produces the statement *“$3 = 0 + 3$,”* which is correct. This is **True ($\text{T}$)**.

#### Textbook Example 4: Routing Connections
Let $A(c, n)$ denote the statement *“Computer $c$ is connected to network $n$.”* Suppose computer `MATH1` is connected to network `CAMPUS2`, but is isolated from network `CAMPUS1`.
* **$A(\text{MATH1}, \text{CAMPUS1})$** evaluates to **False ($\text{F}$)**.
* **$A(\text{MATH1}, \text{CAMPUS2})$** evaluates to **True ($\text{T}$)**.

#### Textbook Example 5: 3-Variable Arithmetic
Let $R(x, y, z)$ denote the statement *“$x + y = z$.”* 
* **$R(1, 2, 3)$** produces *“$1 + 2 = 3$,”* which is **True ($\text{T}$)**.
* **$R(0, 0, 1)$** produces *“$0 + 0 = 1$,”* which is **False ($\text{F}$)**.

### Predicates in Software Engineering: Preconditions & Postconditions

In computer programming, predicates serve as the underlying foundation for **program verification**—the formal process of mathematically proving that a program produces correct outputs for all possible valid inputs, rather than relying on randomized testing.

To achieve this, we use two specific classifications of predicates:
1. **Preconditions:** Predicates that define the required state and valid parameters of the data *before* a code segment runs.
2. **Postconditions:** Predicates that describe the conditions that the output variables must satisfy *after* the execution of the code segment finishes.

#### Textbook Example 7: The Swapping Code Verification
Consider a short code segment designed to swap or interchange the values of two variables, $x$ and $y$:

```text
temp := x
x := y
y := temp
```

Let’s construct the formal verification tracking using predicates from your text:

* **Step 1: Setting up the Precondition**
  Before the swap executes, we assume $x$ and $y$ hold some distinct initial values $a$ and $b$. We declare our precondition using the predicate $P(x, y)$:
  $$\text{Precondition: } P(x, y) \equiv (x = a \land y = b) \text{ is True.}$$

* **Step 2: Tracking Execution Line-by-Line**
  1. `temp := x` $\rightarrow$ The value of $x$ is copied into `temp`. The tracking state becomes: $x = a, y = b, \text{ temp} = a$.
  2. `x := y` $\rightarrow$ The current value of $y$ is copied into $x$. The tracking state becomes: $x = b, y = b, \text{ temp} = a$.
  3. `y := temp` $\rightarrow$ The saved value in `temp` is copied into $y$. The tracking state becomes: $x = b, y = a, \text{ temp} = a$.

* **Step 3: Evaluating the Postcondition**
  Our desired outcome is that $x$ ends up with $b$, and $y$ ends up with $a$. We define this postcondition as $Q(x, y)$:
  $$\text{Postcondition: } Q(x, y) \equiv (x = b \land y = a)$$

Since the line-by-line mechanical execution directly matches the requirements of our postcondition predicate $Q(x, y)$, the program segment is formally and securely proven to be correct for all valid inputs!

## 1.4.3 Quantifiers

When variables are present in a propositional function (like $P(x)$), assigning a specific value to that variable turns it into a valid proposition with a distinct truth value. However, there is another deeply powerful way to turn a propositional function into a concrete proposition: **Quantification**.

> **Definition:** **Quantification** expresses the extent to which a predicate is true over a specific range of elements.

In natural human languages, we constantly use quantification through words like *all, some, many, none,* and *few*. In formal computer science and mathematical logic, we focus heavily on the two primary types of quantification:
1. **Universal Quantification:** Asserts that a predicate is true for *every single element* under consideration.
2. **Existential Quantification:** Asserts that there is *one or more elements* under consideration for which the predicate holds true.

The area of logic that seamlessly blends predicates and quantifiers is officially called the **predicate calculus**.

---

### The Fundamental Rule: The Domain of Discourse

Before you can ever determine the truth value of a quantified statement, you **must explicitly define the domain**. The **domain of discourse** (or universe of discourse, often simply called the **domain**) specifies the exact collection of possible values that the variable $x$ is allowed to take.

* Changing the domain can completely flip the truth value of a quantified statement from True to False!
* Without an explicitly stated or understood domain, a quantified expression is mathematically **undefined** and has no meaning.
* *Standard textbook convention:* We universally assume that all domains of discourse are **nonempty** unless specified otherwise.

---

## 1. The Universal Quantifier ($\forall$)

Many mathematical assertions state that a property is true for *all* values of a variable within its domain. We express this using the **universal quantifier**, denoted by the symbol **$\forall$**.

> **Definition:** The universal quantification of $P(x)$ is the statement: *"P(x) for all values of x in the domain."* It is written symbolically as:
> $$\forall x P(x)$$
> This is read aloud as *"for all x, P(x)"* or *"for every x, P(x)"*.

### What Makes $\forall x P(x)$ True or False?
* **When is it True?** $\forall x P(x)$ is true if and only if $P(x)$ evaluates to True for *every single element* in the domain.
* **When is it False?** It is false if there is *at least one* element in the domain for which $P(x)$ is false.

> **Definition:** An individual element $c$ in the domain for which $P(c)$ is false is called a **counterexample** to the statement $\forall x P(x)$. Providing a **single counterexample** is all that is strictly required to mathematically prove that a universally quantified statement is false.

#### Common Natural Language Phrasings for $\forall$:
Expressions like *"all of,"* *"for each,"* *"given any,"* *"for arbitrary,"* and *"for any"* map directly to the universal quantifier.
* ⚠️ *Textbook Tip:* Be highly careful using the word *"any"* in ambiguous phrasing. It is generally safest to stick to explicitly writing *"for every"* or *"for all"* to prevent confusion.

#### Textbook Universal Quantification Walkthroughs:

* **EXAMPLE 8: Real Numbers Identity**
  Let $P(x)$ be the statement $“x + 1 > x.”$ What is the truth value of $\forall x P(x)$ if the domain consists of all real numbers?
  * *Solution:* Since adding $1$ to any real number always yields a value strictly greater than the original number, $P(x)$ is true for every element in the domain. Therefore, the quantification $\forall x P(x)$ is **True ($\text{T}$)**.

* **EXAMPLE 9: Sifting out Counterexamples**
  Let $Q(x)$ be the statement $“x < 2.”$ What is the truth value of $\forall x Q(x)$ over the domain of all real numbers?
  * *Solution:* $Q(x)$ is not true for *every* real number. For instance, if we select $x = 3$, we get $3 < 2$, which is false. Because $x = 3$ serves as a valid **counterexample**, the statement $\forall x Q(x)$ is **False ($\text{F}$)**.

* **EXAMPLE 10: The Zero Factor**
  Let $P(x)$ be the statement $“x^2 > 0”$ over the domain of all integers.
  * *Solution:* If we select the integer $x = 0$, we find that $0^2 = 0$, which is *not* strictly greater than $0$. Thus, $x = 0$ is a counterexample, rendering $\forall x P(x)$ **False ($\text{F}$)**.

* **EXAMPLE 12: Showing How Domains Dictate Truth**
  What is the truth value of $\forall x (x^2 \ge x)$ if the domain is: (a) all real numbers, or (b) all integers?
  * *Solution for (a):* Over the real numbers, this is **False**. If we choose the decimal fraction $x = 0.5$, we find that $(0.5)^2 = 0.25$. Since $0.25 \ge 0.5$ is completely false, $x = 0.5$ is a valid counterexample. (The inequality fails for all real numbers where $0 < x < 1$).
  * *Solution for (b):* Over the integers, this is **True**. There are absolutely no discrete integers that exist strictly between $0$ and $1$, so the condition holds perfectly for every element in the integer domain.

---

## 2. The Existential Quantifier ($\exists$)

When a statement asserts that there is *at least one* element in the domain possessing a specific trait, we employ the **existential quantifier**, denoted by the symbol **$\exists$**.

> **Definition:** The existential quantification of $P(x)$ is the proposition: *"There exists an element x in the domain such that P(x)."* It is written symbolically as:
> $$\exists x P(x)$$
> This can be read aloud as *"There is an x such that P(x)"* or *"For some x, P(x)"*.

### What Makes $\exists x P(x)$ True or False?
* **When is it True?** $\exists x P(x)$ is true if there is **at least one** element $c$ in the domain for which $P(c)$ evaluates to True.
* **When is it False?** It is false if and only if $P(x)$ is false for *every single element* across the entire domain.

#### Common Natural Language Phrasings for $\exists$:
Phrases like *"for some,"* *"for at least one,"* or *"there is"* map directly to an existential quantifier expression.

#### Textbook Existential Quantification Walkthroughs:

* **EXAMPLE 13: Finding a Single Match**
  Let $P(x)$ denote the statement $“x > 3.”$ What is the truth value of $\exists x P(x)$ if the domain consists of all real numbers?
  * *Solution:* We only need to find a single element that satisfies the condition. For example, when we choose $x = 4$, the statement $4 > 3$ is perfectly true. Therefore, $\exists x P(x)$ evaluates to **True ($\text{T}$)**.

* **EXAMPLE 14: An Impossible Equality**
  Let $Q(x)$ denote the statement $“x = x + 1.”$ What is the truth value of $\exists x Q(x)$ over the domain of all real numbers?
  * *Solution:* No real number can ever equal itself plus one. Because the predicate statement $Q(x)$ evaluates to false for *every single element* in the universe of discourse, the existential quantification $\exists x Q(x)$ is **False ($\text{F}$)**.

### Quantifier Summary Reference

To help you seamlessly memorize how these two operators behave, keep this structural table close at hand:

| Statement | When is it True? | When is it False? |
| :--- | :--- | :--- |
| **$\forall x P(x)$** | $P(x)$ is true for **every** $x$. | There is an $x$ for which $P(x)$ is **false**. |
| **$\exists x P(x)$** | There is an $x$ for which $P(x)$ is **true**. | $P(x)$ is false for **every** $x$. |

## 3. Bonus: The Uniqueness Quantifier ($\exists!$ or $\exists_1$)

While the universal and existential quantifiers are by far the most widely used tools in discrete mathematics and computer science, logic allows us to define custom quantifiers to declare exact counts. The most common specialty extension you will encounter is the **uniqueness quantifier**, written using the symbol **$\exists!$** or **$\exists_1$**.

> **Definition:** The statement **$\exists! x P(x)$** asserts that *"There exists a unique x such that P(x) is true."* 

This can be read contextually as *"there is exactly one"* or *"there is one and only one"* element that fits the description.
* **Example:** The expression $\exists! x (x - 1 = 0)$ over the domain of real numbers is **True**, because $x = 1$ is the *one and only* real number that satisfies that specific arithmetic equation.

> [!NOTE]
> We can actually express uniqueness completely using a combination of existential and universal quantifiers paired with standard propositional logic connectives. Because of this, the uniqueness quantifier is technically optional, and it is usually best practice to stick to standard $\forall$ and $\exists$ statements when you are building proofs.
