# 1.1.4 Truth Tables of Compound Propositions

Now that we have covered the five main logical connectives—**negation ($\neg$)**, **conjunction ($\land$)**, **disjunction ($\lor$)**, **exclusive or ($\oplus$)**, **implication ($\rightarrow$)**, and **biconditional ($\leftrightarrow$)**—we have the tools to build complex **compound propositions** involving any number of variables.

To determine the final truth value of these complicated expressions under all possible conditions, we use a structured methodology known as a **truth table**.

---

### The Rules of Construction

When constructing a truth table for a large expression, you shouldn't try to compute the final answer all at once. Instead, break it down using separate columns:

1. **Initial Columns:** Create a baseline column for each basic propositional variable involved.
2. **Intermediate Columns:** Step-by-step, create a column for each sub-expression as the compound proposition is built up from its atomic parts.
3. **Final Column:** The final column displays the definitive truth values of the entire compound expression.

#### How many rows do you need?

The number of rows in your truth table is strictly determined by the number of unique variables present in the statement.
* For $n$ propositional variables, you need **$2^n$ rows**.
* Every time you introduce a new variable, the size of the table doubles.
* **1 variable ($p$):** $2^1 = 2$ rows.
* **2 variables ($p, q$):** $2^2 = 4$ rows (TT, TF, FT, FF).
* **3 variables ($p, q, r$):** $2^3 = 8$ rows.
* **4 variables ($p, q, r, s$):** $2^4 = 16$ rows.

### Comprehensive Textbook Walkthrough: Example 14

Let's carefully construct the truth table for the following compound proposition:

$$(p \lor \neg q) \rightarrow (p \land q)$$

#### Step 1: Analyze the Variables and Determine Rows
The proposition contains **two** variables: $p$ and $q$. Therefore, our table will require $2^2 = 4$ rows to exhaustively cover all possible combinations of inputs: TT, TF, FT, and FF.

#### Step 2: Establish the Structural Breakdown
To build our intermediate columns, we look at the components of the expression:
* We need $\neg q$ to complete the left parenthetical block.
* We need a column for the left block: $(p \lor \neg q)$.
* We need a column for the right parenthetical block: $(p \land q)$.
* Finally, we need the column linking them together via implication: $(p \lor \neg q) \rightarrow (p \land q)$.

#### Step 3: Compute Row by Row

Let's evaluate how each row computes across our columns:

| $p$ | $q$ | $\neg q$ | $p \lor \neg q$ | $p \land q$ | $(p \lor \neg q) \rightarrow (p \land q)$ |
| :---: | :---: | :---: | :---: | :---: | :---: |
| **T** | **T** | F | T | T | **T** |
| **T** | **F** | T | T | F | **F** |
| **F** | **T** | F | F | F | **T** |
| **F** | **F** | T | T | F | **F** |

* **Row 1 (T, T):** $\neg q$ is **F**. For the disjunction $(p \lor \neg q)$, since $p$ is True, it evaluates to **T**. For the conjunction $(p \land q)$, both are True, evaluating to **T**. Finally, the implication $T \rightarrow T$ results in **T**.
* **Row 2 (T, F):** $\neg q$ becomes **T**. The disjunction $(p \lor \neg q)$ evaluates to **T**. The conjunction $(p \land q)$ evaluates to **F**. This leaves us with an implication of $T \rightarrow F$, which violates our contract and evaluates to **F**.
* **Row 3 (F, T):** $\neg q$ is **F**. Since both components of $(p \lor \neg q)$ are False, it evaluates to **F**. The conjunction $(p \land q)$ is also **F**. The resulting implication is $F \rightarrow F$, which is **T** by default.
* **Row 4 (F, F):** $\neg q$ is **T**. The disjunction $(p \lor \neg q)$ catches the true value and evaluates to **T**. The conjunction $(p \land q)$ is **F**. The final implication is $T \rightarrow F$, which evaluates to **F**.

### Step-by-Step 3-Variable Layout: 8 Rows

To ensure you can manage a larger layout seamlessly, let's look at how to set up the baseline variables for an 8-row table ($p, q, r$).

To avoid accidentally skipping or repeating a combination, always use this binary down-counting pattern:
* **Column $p$:** Alternates 4 True, 4 False (TTTT FFFF)
* **Column $q$:** Alternates 2 True, 2 False (TTFF TTFF)
* **Column $r$:** Alternates 1 True, 1 False (TFTF TFTF)

```text
Row 1: T T T
Row 2: T T F
Row 3: T F T
Row 4: T F F
Row 5: F T T
Row 6: F T F
Row 7: F F T
Row 8: F F F
```

### 1.1.5 How the Logical Mnemonic Works

A **mnemonic** (pronounced *neh-MON-ik*) is a system or memory tool—like a pattern of letters, ideas, or associations—that helps you easily recall information. For example, **$\text{PEMDAS}$** is a classic letter mnemonic used to remember the order of operations in arithmetic: **P**arentheses, **E**xponents, **M**ultiplication and **D**ivision (from left to right), **A**nd addition and **S**ubtraction (from left to right).

In **Discrete Mathematics**, we have a matching structural hierarchy for logical operators so that complex expressions can be read unambiguously. In our textbook, we can use a specialized mnemonic or shorthand to memorize how logical statements get parsed by a computer or mathematician when parentheses are missing.

| Priority | Operator Symbol | Name | How it works / What it means / Algebraic Analog |
| :---: | :---: | :---: | --- |
| **1** | $\neg$ | **Negation (NOT)** | Always applies first to the variable directly next to it. (Negative signs/Unary minus: $-x \times y \equiv (-x) \times y$). |
| **2** | $\land$ | **Conjunction (AND)** | Binds tightly like arithmetic multiplication ($\times$). |
| **3** | $\lor$ | **Disjunction (OR)** | Binds more loosely, like arithmetic addition ($+$). |
| **4** | $\rightarrow$ | **Conditional (Implication)** | Evaluates after the core arithmetic operations are completed. (Relational operators: $<, \le$). |
| **5** | $\leftrightarrow$ | **Biconditional (IFF)** | Evaluates dead last in the entire expression. (Equality: $=$). |

#### **Why Conjunction ($\land$) acts like Multiplication:**
Notice how in Table 6 of your text, the Distributive Law shows that $p \land (q \lor r) \equiv (p \land q) \lor (p \land r)$. This mirrors basic algebra perfectly: $x \times (y + z) = (x \times y) + (x \times z)$. Because of this structural tie, **$\land$ always binds more tightly than $\lor$**.

### Processing an Expression Step-by-Step

Let's take the following compound proposition and look at how a computer interprets it line-by-line using these exact priority rules:

$$\neg p \land q \rightarrow r$$

Because there are **3 distinct variables** ($p, q,$ and $r$), any truth table built for this would require exactly $2^3 = 8$ rows to solve completely.

#### Step 1: Negation First ($\neg$)
Negation has the highest priority (Level 1). It locks onto $p$ immediately.
* *Computer reads it as:* $(\neg p) \land q \rightarrow r$

#### Step 2: Conjunction Next ($\land$)
Conjunction has a higher priority than the conditional (Level 2 vs Level 4). It groups the left side together.
* *Computer reads it as:* $((\neg p) \land q) \rightarrow r$

#### Step 3: Conditional Last ($\rightarrow$)
The implication has the lowest priority here (Level 4), so it processes the final bridge after everything else is fully resolved.
* *Computer reads it as:* $(((\neg p) \land q) \rightarrow r)$
