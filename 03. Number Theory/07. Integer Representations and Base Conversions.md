# Integer Representations and Base Conversions (Section 4.2.2)

---

Integers can be expressed using any integer greater than one as a base. While we commonly use decimal (base 10) in everyday life, computer science relies heavily on binary (base 2), octal (base 8), and hexadecimal (base 16) representations because of how digital hardware is constructed.

---

### 1. The Theory of Base Expansions

In standard decimal notation, an integer $n$ is structured as a sum of coefficients multiplied by powers of 10:
$$n = a_k10^k + a_{k-1}10^{k-1} + \dots + a_110 + a_0$$
where each $a_j$ is an integer digit between 0 and 9. 

Theorem 1 establishes that this structure is valid for *any* positive integer base greater than 1:

> **Theorem 1:** Let $b$ be an integer greater than 1. If $n$ is a positive integer, it can be expressed uniquely in the form:
> $$n = a_kb^k + a_{k-1}b^{k-1} + \dots + a_1b + a_0$$
> where $k$ is a nonnegative integer, $a_0, a_1, \dots, a_k$ are nonnegative integers less than $b$, and $a_k \neq 0$.

* **Notation:** The representation of $n$ in base $b$ is written as $(a_ka_{k-1}\dots a_1a_0)_b$.
* When no subscript is present, base 10 (decimal) is assumed.

#### Common Computer Science Bases:
1. **Binary (Base 2):** Digits (bits) are strictly $0$ or $1$. A binary expansion is a bit string processed directly by computer registers.
2. **Octal (Base 8):** Uses digits $0$ through $7$.
3. **Hexadecimal (Base 16):** Uses 16 distinct digits: $0$–$9$ for values 0–9, and the uppercase letters $\text{A, B, C, D, E, F}$ to represent values 10, 11, 12, 13, 14, and 15, respectively.

---

### 2. Conversions to Decimal (Base 10)

To convert an integer from base $b$ to base 10, simply expand the positional polynomial and compute the sum using normal decimal arithmetic.

* **Example 1 (Binary to Decimal):**
  $$(101011111)_2 = 1\cdot2^8 + 0\cdot2^7 + 1\cdot2^6 + 0\cdot2^5 + 1\cdot2^4 + 1\cdot2^3 + 1\cdot2^2 + 1\cdot2^1 + 1\cdot2^0 = 351$$

* **Example 2 (Octal to Decimal):**
  $$(7016)_8 = 7\cdot8^3 + 0\cdot8^2 + 1\cdot8^1 + 6\cdot8^0 = 3598$$

* **Example 3 (Hexadecimal to Decimal):**
  $$(2\text{AE0B})_{16} = 2\cdot16^4 + 10\cdot16^3 + 14\cdot16^2 + 0\cdot16^1 + 11\cdot16^0 = 175627$$

---

### 3. Converting Decimal to Base $b$

To convert a decimal integer $n$ into base $b$, we successively divide $n$ and its quotients by $b$, collecting the remainders. The remainders generated at each step yield the digits of the base $b$ expansion from right to left (least significant to most significant digit).

#### Pseudocode:
```
procedure base_b_expansion(n, b: positive integers with b > 1)
    q := n
    k := 0
    while q ≠ 0
        a_k := q mod b   (this remainder provides the next digit)
        q := q div b     (this updates the quotient for the next loop)
        k := k + 1
    return (a_{k-1}, ..., a_1, a_0)
```

* **Example 4 (Decimal to Octal):** Find the octal expansion of $12345_{10}$.
  * $12345 = 8 \cdot 1543 + \mathbf{1}$ (Remainder $a_0 = 1$)
  * $1543 = 8 \cdot 192 + \mathbf{7}$ (Remainder $a_1 = 7$)
  * $192 = 8 \cdot 24 + \mathbf{0}$ (Remainder $a_2 = 0$)
  * $24 = 8 \cdot 3 + \mathbf{0}$ (Remainder $a_3 = 0$)
  * $3 = 8 \cdot 0 + \mathbf{3}$ (Remainder $a_4 = 3$)
  * Reading the remainders from bottom to top yields: $(30071)_8$.

* **Example 5 (Decimal to Hexadecimal):** Find the hexadecimal expansion of $177130_{10}$.
  * $177130 = 16 \cdot 11070 + \mathbf{10} \rightarrow \mathbf{A}$
  * $11070 = 16 \cdot 691 + \mathbf{14} \rightarrow \mathbf{E}$
  * $691 = 16 \cdot 43 + \mathbf{3}$
  * $43 = 16 \cdot 2 + \mathbf{11} \rightarrow \mathbf{B}$
  * $2 = 16 \cdot 0 + \mathbf{2}$
  * Result: $(2\text{B3EA})_{16}$.

---

### 4. Short-Cut Conversions: Binary, Octal, and Hexadecimal

Because $8 = 2^3$ and $16 = 2^4$, conversions between binary, octal, and hexadecimal do not require arithmetic operations. Instead, digits can be directly mapped using bit blocks.

* **Binary to Octal (3-bit blocks):** Group bits into blocks of 3 starting from the right. Translate each block to its octal digit.
* **Binary to Hexadecimal (4-bit blocks):** Group bits into blocks of 4 starting from the right (pad with leading zeros if necessary).
  * Given $(11\,1110\,1011\,1100)_2 \rightarrow$ split into: `0011` `1110` `1011` `1100`.
  * Translate: $0011_2 = 3$, $1110_2 = \text{E}$, $1011_2 = \text{B}$, $1100_2 = \text{C}$.
  * Result: $(3\text{EBC})_{16}$.

* **Hexadecimal to Binary:** Replace every single hex character with its corresponding 4-bit binary layout.
  * $(\text{A8D})_{16} \rightarrow \text{A} = 1010$, $8 = 1000$, $\text{D} = 1101 \rightarrow (101010001101)_2$.

### 🧠 Check for Understanding

1. Convert $(110111)_2$ to decimal.
2. Convert $241_{10}$ to binary using the expansion algorithm.
3. Convert $(3\text{C}5)_{16}$ directly to binary.

---

### 💡 Solutions & Explanation

> [!NOTE]
> Here are the step-by-step conversions:
> 
> 1. **$(110111)_2$ to Decimal:**
>    * $1\cdot 2^5 + 1\cdot 2^4 + 0\cdot 2^3 + 1\cdot 2^2 + 1\cdot 2^1 + 1\cdot 2^0$
>    * $= 32 + 16 + 0 + 4 + 2 + 1 = 55$.
> 
> 2. **$241_{10}$ to Binary:**
>    * $241 = 2 \cdot 120 + \mathbf{1}$
>    * $120 = 2 \cdot 60 + \mathbf{0}$
>    * $60 = 2 \cdot 30 + \mathbf{0}$
>    * $30 = 2 \cdot 15 + \mathbf{0}$
>    * $15 = 2 \cdot 7 + \mathbf{1}$
>    * $7 = 2 \cdot 3 + \mathbf{1}$
>    * $3 = 2 \cdot 1 + \mathbf{1}$
>    * $1 = 2 \cdot 0 + \mathbf{1}$
>    * Result: $(11110001)_2$.
> 
> 3. **$(3\text{C}5)_{16}$ directly to Binary:**
>    * $3 \rightarrow 0011$
>    * $\text{C} \rightarrow 1100$
>    * $5 \rightarrow 0101$
>    * Strip leading zeros: $(1111000101)_2$.
