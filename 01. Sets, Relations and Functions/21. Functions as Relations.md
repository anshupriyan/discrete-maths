# Functions as Relations (Section 9.1.2)

---

### 1. The Core Graph Connection

A function $f$ from a set $A$ to a set $B$ assigns **exactly one** element of $B$ to each element of $A$. The **graph** of that function gathers all the ordered pairs $(a, b)$ such that $b = f(a)$.

Because the graph of $f$ is a collection of ordered pairs where the first entry is from $A$ and the second is from $B$, it is a subset of the Cartesian product $A \times B$. By definition, **any subset of $A \times B$ is a relation**.

Therefore:
> Every function dynamically determines a binary relation via its graph.

---

### 2. When is a Relation a Function?

Since relations are broader structures that allow many-to-many pairings, not all relations are functions. The textbook provides a clear rule for converting a relation back into a function:

> **The Function Condition:** A relation $R$ from $A$ to $B$ is the graph of a function if and only if **every element in $A$ is the first element of exactly one ordered pair in $R$**.

If this condition is met, we can safely define a corresponding function by assigning $f(a) = b$ for the unique pair $(a, b) \in R$.

---

### 3. Textbook Analysis: Why Certain Relations Fail the Function Test

#### **The U.S. Cities Example**
Recall the U.S. Cities relation, where $A$ is the set of U.S. cities and $B$ is the set of the 50 states. The relation $R$ pairs a city $a$ with a state $b$ if that city is located in that state.
* Pairs in this relation include: `(Middletown, New Jersey)` and `(Middletown, New York)`.

**Textbook Finding:** This relation is **not** the graph of a function. Because the city name `Middletown` occurs *more than once* as the first element of an ordered pair, it maps to multiple states. A function cannot have a single input mapping to multiple outputs.

#### **Structural Comparison Summary**
The conceptual differences between relations and functions are summarized below:
* A **relation** can express a *one-to-many* relationship between elements of two sets, where an element of $A$ can link to zero, one, or many items in $B$.
* A **function** strictly represents a relation where *exactly one* element of $B$ is related to each element of $A$.

Thus, relations are a broad **generalization** of functions, allowing us to describe a much wider class of interactions between sets.

### 🧠 Quick Check: Try it Yourself!

Let $A = \{a, b, c\}$ and $B = \{1, 2, 3\}$. Identify which of the following relations from $A$ to $B$ represent a valid function:
1. $R_1 = \{(a, 2), (b, 1), (c, 3)\}$
2. $R_2 = \{(a, 1), (b, 2)\}$
3. $R_3 = \{(a, 1), (b, 2), (c, 3), (a, 3)\}$

---

### 💡 Solutions & Explanation

> [!NOTE]
> Here are the step-by-step verification answers for the check above:
> 
> 1. **$R_1$:** **Valid Function**.
>    * *Proof:* Every element of the domain $A = \{a, b, c\}$ appears as the first element of exactly one ordered pair in $R_1$ ($a \mapsto 2$, $b \mapsto 1$, $c \mapsto 3$). It satisfies the Function Condition.
> 2. **$R_2$:** **Not a Valid Function**.
>    * *Proof:* The element $c \in A$ does not appear as the first element of any ordered pair in $R_2$. By definition, a function must assign an output to *every* element of the domain.
> 3. **$R_3$:** **Not a Valid Function**.
>    * *Proof:* The element $a \in A$ appears as the first element of two different ordered pairs: $(a, 1)$ and $(a, 3)$. This means the single input $a$ is mapped to two different outputs ($1$ and $3$), which violates the definition of a function.
