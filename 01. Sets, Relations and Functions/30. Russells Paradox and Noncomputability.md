# Russell's Paradox and Noncomputability (Section 2.1 - 2.2)

---

When we specify a set, we usually do it in one of two ways:
1. **Roster Method:** Exhaustively listing every element (e.g., $\{9, 16, 25\}$).
2. **Set-Builder Notation:** Specifying a property that uniquely characterizes the elements (e.g., $\{x \mid x \text{ is a perfect square, } 5 \le x \le 50\}$).

While set-builder notation is incredibly useful for large or infinite sets, the early logician Bertrand Russell discovered a massive logical pitfall in 1901 when you allow *any* arbitrary property to define a set (referred to as **naive set theory**).

---

### 1. The Mathematical Paradox

Consider the set $S$ defined as:
$$S = \{x \mid x \notin x\}$$

In plain English, $S$ is the **"set of all sets that do not contain themselves."** Now, we ask a seemingly simple question: **Is $S$ an element of $S$ ($S \in S$)?**

* **Assumption 1: $S \in S$**
  * If $S$ is in $S$, it must satisfy the property defining $S$. The property is $x \notin x$. Therefore, $S \notin S$. This is a direct contradiction.
* **Assumption 2: $S \notin S$**
  * If $S$ is not in $S$, then it satisfies the condition to be a member of $S$. Therefore, $S \in S$. This is also a direct contradiction.

Because both paths lead to a logical contradiction, the way we specified the set is fundamentally broken. To avoid this paradox, modern mathematics uses **axiomatic set theory** (such as Zermelo-Fraenkel set theory), which restricts how sets can be formed.

---

### 2. Connection to Computer Science: The Halting Problem

In theoretical computer science, Russell's Paradox is closely mirrored by Alan Turing's **Halting Problem**. 

The Halting Problem proves that it is impossible to write a program $T$ that takes any program $P$ and input $I$ and perfectly decides whether $P$ will loop infinitely or stop (halt). 

The proof uses self-reference and negation in the exact same way as Russell's Paradox: if you construct a program that halts when the test program loops, and loops when the test program halts, running this program on *itself* creates a logical contradiction.

---

### 3. Classic Analogies of the Paradox

#### A. The Barber Paradox
A barber in a village shaves *only* those men who do not shave themselves. Does the barber shave himself?
* If he shaves himself, he violates the rule (he shaves someone who shaves himself).
* If he does not shave himself, he violates the rule (he must shave himself since he shaves those who don't shave themselves).
* **Conclusion:** No such barber can exist.

#### B. Heterological Words
An adjective is **heterological** if it does not describe itself. For example, "monosyllabic" is heterological because it has 5 syllables (meaning it is not monosyllabic). Conversely, "short" is autological (describes itself).
* Is the word "heterological" heterological?
* If it is, then it describes itself, meaning it is autological (not heterological).
* If it is not, then it does not describe itself, meaning it must be heterological.
* **Conclusion:** This property leads to a direct self-referential contradiction.
