# Introduction to Relations (Sections 9.1 - 9.1.1)

---

### 1. The Core Motivation: Connecting Objects

In discrete mathematics and computer science, we frequently model associations between elements of different sets:
* A business associates an employee with their telephone extension.
* A city associates a street with its postal zip code.
* A compiler associates a variable name with its memory address.
* In a database, table entries are linked across keys (e.g., student IDs to enrolled courses).

Previously, **functions** were used to map an element from a domain to exactly one element in a codomain. However, functions have a strict limitation: *an input can only map to exactly one output.* 

To model many-to-many connections—such as a single student enrolled in multiple courses, or a single web server handling multiple clients—we loosen this restriction and use **relations**.

---

### 2. Formal Mathematical Definition

A relation is defined mathematically as a set of ordered pairs. This is called a **binary relation** because it establishes a link between elements of two sets.

> **Definition 1:** Let $A$ and $B$ be nonempty sets. A **binary relation from $A$ to $B$** is a subset of the Cartesian product $A \times B$.

* **Notation:** A binary relation is typically denoted by the capital letter **$R$**.
* If an ordered pair $(a, b)$ belongs to the relation $R$, we write **$a \, R \, b$** (read as "$a$ is related to $b$ by $R$").
* If $(a, b)$ does not belong to $R$, we write **$a \not R \, b$** ("$a$ is not related to $b$ by $R$").

#### **Key Distinction: Relations vs. Functions**
Every function is a relation, but a highly restricted one: each element $a$ in the domain must appear as the first entry in *exactly one* ordered pair. A standard relation is unrestricted: an element $a$ can map to zero, one, or multiple elements in $B$.

### 3. Textbook Examples

#### **Textbook Example 1:**
Let $A$ be the set of students $\{\text{Anish}, \text{Belinda}, \text{Charles}\}$ and let $B$ be the set of courses $\{\text{CS101}, \text{CS201}, \text{Math211}\}$.

Suppose that Anish is enrolled in CS101 and Math211; Belinda is enrolled in CS201 and Math211; and Charles is enrolled in CS101 and CS201. Express this enrollment as a relation $R$.

* **Solution:** We collect these specific pairings into a set of ordered pairs:
  $$\mathbf{R = \{(\text{Anish}, \text{CS101}), (\text{Anish}, \text{Math211}), (\text{Belinda}, \text{CS201}), (\text{Belinda}, \text{Math211}), (\text{Charles}, \text{CS101}), (\text{Charles}, \text{CS201})\}}$$

### 4. Relations on a Single Set

In computer science, webpage graphs, and social networks, we frequently relate elements of a set to other elements *within that same set* (e.g., webpages linking to webpages, or people related to siblings).

> **Definition 2:** A relation on a set $A$ is a relation from $A$ to $A$ (in other words, it is a subset of the Cartesian product $A \times A$).

#### **Textbook Example 3:**
Let $A = \{1, 2, 3, 4\}$. Which ordered pairs are in the relation $R = \{(a, b) \mid a \text{ divides } b\}$?

* **Solution:** The notation "$a$ divides $b$" (written $a \mid b$) means that $b/a$ results in an integer with a remainder of zero. We test all elements in $A$:
  * **For $a = 1$:** 1 divides all integers $\rightarrow (1,1), (1,2), (1,3), (1,4)$
  * **For $a = 2$:** 2 divides 2 and 4 $\rightarrow (2,2), (2,4)$
  * **For $a = 3$:** 3 divides 3 $\rightarrow (3,3)$
  * **For $a = 4$:** 4 divides 4 $\rightarrow (4,4)$
* Putting it all together, our relation is:
  $$\mathbf{R = \{(1,1), (1,2), (1,3), (1,4), (2,2), (2,4), (3,3), (4,4)\}}$$

### 🧠 Quick Check: Try it Yourself!

Let $A = \{1, 2, 3\}$. Consider a relation $R$ on the set $A$ defined by the rule:
$$R = \{(a, b) \mid a < b\}$$

List all the ordered pairs that belong to this relation $R$.

---

### 💡 Solutions & Explanation

> [!NOTE]
> Here are the step-by-step verification answers for the check above:
> 
> * **Answer:** **$\{(1, 2), (1, 3), (2, 3)\}$**.
> * **Proof:** We systematically evaluate all possible ordered pairs $(a, b)$ in $A \times A$ to see if they satisfy the condition $a < b$:
>   * For $a = 1$: $1 < 2$ (Yes) and $1 < 3$ (Yes) $\rightarrow (1, 2), (1, 3)$
>   * For $a = 2$: $2 < 3$ (Yes) $\rightarrow (2, 3)$
>   * For $a = 3$: $3$ is not strictly less than any element in $A$ (No)
>   Any other pair (such as $(2, 1)$ or $(3, 2)$) fails the inequality because $a \ge b$.
