# Recurrence Relations and Modeling (Section 10.1 - 10.2)

---

A recurrence relation is a way of defining a sequence where each term is expressed in terms of one or more of its preceding terms. In computer science, recurrence relations are the standard tool to represent the cost of recursive algorithms, count combinations, and model population growth.

---

### 1. Core Definitions

> **Definition 1:** A **recurrence relation** for a sequence $a_0, a_1, a_2, \dots$ is an equation that expresses the $n$-th term $a_n$ in terms of one or more of its predecessors $a_{0}, a_{1}, \dots, a_{n-1}$, for all integers $n$ greater than or equal to a certain index.

* **Initial Conditions:** The starting values of the sequence that are specified to uniquely determine all subsequent terms. A recurrence of order $k$ requires exactly $k$ initial conditions.
* **Order (or Delay):** The difference between the highest and lowest indices in the recurrence relation. For example, $a_n = 3a_{n-1} + 2a_{n-2}$ has order $2$ (since the indices differ by $n - (n-2) = 2$).
* **Linear Recurrence:** A recurrence relation is **linear** if every term $a_i$ appears to the power 1, and there are no products of terms (e.g., $a_{n-1} \cdot a_{n-2}$ is forbidden).
  * *Linear:* $a_n = 2a_{n-1} + a_{n-2}$
  * *Non-Linear:* $a_n = (a_{n-1})^2 + a_{n-2}$ or $a_n = a_{n-1}a_{n-2}$

### 2. Modeling Classic Problems

Below are classic problems that are modeled using recurrence relations:

#### A. Fibonacci Numbers
The Fibonacci sequence models population growth (specifically, breeding rabbits under simplified conditions).
* **Initial Conditions:** $f_0 = 0, \quad f_1 = 1$
* **Recurrence Relation:**
  $$f_n = f_{n-1} + f_{n-2} \quad (\text{for } n \ge 2)$$
* **Sequence:** $0, 1, 1, 2, 3, 5, 8, 13, 21, \dots$

#### B. The Towers of Hanoi
The puzzle consists of $n$ disks of different sizes placed on one of three pegs in decreasing order of size. The goal is to move the entire stack to another peg under the rules:
1. Only one disk can be moved at a time.
2. A disk can only be placed on a larger disk or an empty peg.

Let $H_n$ be the minimum number of moves to transfer $n$ disks.
* To move $n$ disks:
  1. Move the top $n-1$ disks to the spare peg (takes $H_{n-1}$ moves).
  2. Move the largest disk to the destination peg (takes $1$ move).
  3. Move the $n-1$ disks from the spare peg to the destination peg (takes $H_{n-1}$ moves).
* **Initial Condition:** $H_1 = 1$
* **Recurrence Relation:**
  $$H_n = 2H_{n-1} + 1 \quad (\text{for } n \ge 2)$$

#### C. Counting Bit Strings without Consecutive 0s
Let $a_n$ be the number of bit strings of length $n$ that do not contain consecutive $0$s.
* **Analysis:** A valid string of length $n$ can end in either a `1` or a `0`:
  * If the string ends in `1`, the first $n-1$ bits can be any valid string of length $n-1$. There are $a_{n-1}$ such strings.
  * If the string ends in `0`, the preceding bit must be `1` (to avoid consecutive `0`s). Thus, the string must end in `10`, and the first $n-2$ bits can be any valid string of length $n-2$. There are $a_{n-2}$ such strings.
* **Initial Conditions:**
  * $a_1 = 2$ (strings: `0`, `1`)
  * $a_2 = 3$ (strings: `01`, `10`, `11` - `00` is invalid)
* **Recurrence Relation:**
  $$a_n = a_{n-1} + a_{n-2} \quad (\text{for } n \ge 3)$$

#### D. Compound Interest
Let $P$ be the principal amount invested at an annual interest rate $r$, compounded annually. Let $I_n$ be the value of the investment after $n$ years.
* **Initial Condition:** $I_0 = P$
* **Recurrence Relation:**
  $$I_n = I_{n-1} + r I_{n-1} = (1 + r)I_{n-1} \quad (\text{for } n \ge 1)$$

### 🧠 Check for Understanding

1. Determine the order and linearity of the following recurrence relations:
   * **Relation A:** $a_n = n \cdot a_{n-1} + a_{n-3}$
   * **Relation B:** $a_n = a_{n-1}^3 + 2a_{n-2}$
   * **Relation C:** $a_n = a_{n-1}a_{n-2} + 5$

2. Find the number of valid bit strings of length 5 that do not contain consecutive 0s.

---

### 💡 Solutions & Explanation

> [!NOTE]
> Here are the step-by-step solutions:
> 
> 1. **Order and Linearity:**
>    * **Relation A:** Order is $3$ (highest index $n$, lowest $n-3$). It is **linear** because the terms $a_{n-1}$ and $a_{n-3}$ are to the power 1 (the variable coefficient $n$ does not violate linearity of the sequence terms).
>    * **Relation B:** Order is $2$ (highest index $n$, lowest $n-2$). It is **non-linear** because $a_{n-1}$ is raised to the power 3.
>    * **Relation C:** Order is $2$. It is **non-linear** because it contains the product of sequence terms $a_{n-1}a_{n-2}$.
> 
> 2. **Bit Strings calculation:**
>    * Using the recurrence $a_n = a_{n-1} + a_{n-2}$ with $a_1 = 2$ and $a_2 = 3$:
>      * $a_3 = a_2 + a_1 = 3 + 2 = 5$
>      * $a_4 = a_3 + a_2 = 5 + 3 = 8$
>      * $a_5 = a_4 + a_3 = 8 + 5 = 13$
>    * There are exactly **13** valid bit strings of length 5.
