# Shortest-Path Problems and Dijkstra (Section 10.6.1)

---

Many practical problems can be modeled using weighted graphs, where edges have associated numerical values (e.g., distance, cost, time). A central problem in network routing, GPS navigation, and logistics is finding the path of minimum total weight between two vertices.

---

### 1. Weighted Graphs

> **Definition 1:** A **weighted graph** is a graph $G = (V, E)$ together with a weight function $w: E \to \mathbb{R}^+$ that assigns a positive real number $w(e)$ to each edge $e \in E$. The **length of a path** in a weighted graph is the sum of the weights of the edges in the path.

---

### 2. Dijkstra's Algorithm

**Dijkstra's Algorithm** is an iterative algorithm that finds the shortest path from a single source vertex $a$ to all other vertices in a connected weighted graph with **non-negative edge weights**.

#### Algorithm Logic:
1. **Initialize Labels:** Let $L(v)$ be the label of vertex $v$, representing the current shortest distance from the source $a$ to $v$. Set $L(a) = 0$, and $L(v) = \infty$ for all other vertices $v \ne a$.
2. **Track Visited Set:** Let $S$ be the set of vertices whose shortest paths from $a$ have already been determined. Initially, $S = \emptyset$.
3. **Iterate:** While the destination vertex is not in $S$ (or until all vertices are added to $S$):
   * Select a vertex $u \notin S$ with the minimum label $L(u)$.
   * Add $u$ to $S$ (mark it as visited).
   * For each neighbor $v$ of $u$ that is not in $S$, update its label:
     $$L(v) = \min(L(v), L(u) + w(u, v))$$

---

### 3. Step-by-Step Worked Example

Let's find the shortest path from source vertex **$a$** to all other vertices in the weighted graph below:

* **Vertices ($V$):** $\{a, b, c, d, z\}$
* **Weighted Edges ($E$):**
  * $w(a, b) = 4, \quad w(a, c) = 2$
  * $w(b, c) = 1, \quad w(b, d) = 5$
  * $w(c, d) = 8, \quad w(c, z) = 10$
  * $w(d, z) = 2$

* Path to $b$: $a - c - b$ (Length: $3$)
* Path to $c$: $a - c$ (Length: $2$)

* Path to $z$: $a - c - b - d - z$ (Length: $10$)

### đź§  Check for Understanding

1. Why does Dijkstra's algorithm fail when there are negative edge weights?
2. Find the shortest path from $a$ to $z$ in a graph where:
   * $w(a, b) = 3, \quad w(a, c) = 4$
   * $w(b, z) = 6, \quad w(c, z) = 3$

---

### đź’ˇ Solutions & Explanation

> [!NOTE]
> Here are the solutions:
> 
> 1. **Failure with Negative Weights:**
>    * Dijkstra's algorithm is a **greedy algorithm**. Once a vertex is added to the visited set $S$, its shortest path is assumed to be finalized, and it is never updated again. 
>    * If there are negative edge weights, a path that goes through a vertex added later could end up being shorter than the path already found. For example, if we have $w(a, b) = 2$, $w(a, c) = 3$, and $w(c, b) = -2$, Dijkstra will select $b$ first (since $L(b)=2 < L(c)=3$) and finalize $L(b)=2$. However, the path $a - c - b$ has a smaller weight of $3 + (-2) = 1$.
> 
> 2. **Shortest Path Calculation:**
>    * Path 1: $a - b - z$ has length $3 + 6 = 9$.
>    * Path 2: $a - c - z$ has length $4 + 3 = 7$.
>    * The shortest path is **$a - c - z$** with length **$7$**.
